Descripción del Curso

Interacción Hombre-Computadora (HCI) introduce a los estudiantes a los principios y teorías clave del diseño de interfaces centradas en el usuario

Conceptos Clave
  • Usabilidad y accesibilidad
  • Cognición distribuida
  • Leyes de Fitts y Hick
  • Principios de Gestalt
Tecnologías Emergentes
  • Realidad aumentada y virtual
  • Computación afectiva
  • Interfaces cerebro-computadora
  • IoT y sistemas interactivos

Enfoque práctico: Desarrollo de proyectos colaborativos y análisis de casos reales

Competencias a Desarrollar

Al finalizar la asignatura, el estudiante estará en capacidad de:

Fundamentos

Comprender principios fundamentales de HCI, usabilidad y diseño centrado en el usuario

Aplicación

Aplicar técnicas de diseño basadas en leyes cognitivas y principios de Gestalt

Evaluación

Evaluar interfaces usando heurísticas de Nielsen y Shneiderman

Desarrollo

Desarrollar prototipos con tecnologías emergentes (AR, VR, IoT)

Investigación: Analizar tendencias en HCI, incluyendo interfaces cerebro-computadora y computación afectiva

¿Qué es HCI?

Definición Esencial

"Campo interdisciplinario que se preocupa por el diseño, evaluación e implementación de sistemas computacionales interactivos para uso humano"

— MacKenzie (2013)

Los Tres Elementos Fundamentales

Humanos

Usuarios únicos o múltiples, con habilidades diversas

Ordenadores

No solo PC: dispositivos con chips programables

Interacción

Comunicación bidireccional mediante diversas modalidades

Principios de Usabilidad

Primera Norma de Krug

"¡No me hagas pensar!"

El usuario debe poder mirar una página y entender de qué se trata y cómo usarla sin esfuerzo cognitivo

Clave para PE-1.1: Identificar violaciones a este principio

Ejemplo: "Hace Pensar"

UNIVERSIDAD INTERNACIONAL DEL ECUADOR - SISTEMA ACADÉMICO
Inicio | Académico | Estudiantes | Docentes | Admin | Servicios | Ayuda | Contacto
IMPORTANTE: Seleccione el portal según su modalidad de estudio
Los estudiantes de pregrado deben usar el portal presencial. Los de posgrado el sistema virtual. Para acceso a calificaciones usar la plataforma híbrida. Los docentes tienen su propio acceso...

Problemas Identificados

¿Por qué este diseño "hace pensar"?

  • Múltiples opciones sin diferenciación clara
  • Terminología técnica confusa
  • Usuario debe adivinar cuál elegir
  • Sobrecarga cognitiva innecesaria

Resultado: El usuario pierde tiempo y se frustra antes de completar su tarea

Ejemplo: "Autoexplicativo"

Ventajas del Diseño:

  • Una sola opción de acceso clara
  • Campos etiquetados específicamente
  • Ayuda contextual visible
  • Flujo intuitivo y sin confusión

Reglas de Oro de Shneiderman

Las 8 Reglas Fundamentales

Ben Shneiderman propuso 8 principios fundamentales

Para PE-1.1: Enfócate en Consistencia, Feedback y Prevención de Errores

1. Consistencia

"Esfuérzate por la consistencia"

La regla más frecuentemente violada

Buena Consistencia
Mismo estilo, color y tamaño
  • Botones del mismo color y forma
  • Navegación en la misma posición
  • Terminología uniforme
Inconsistencia
Delete
Diferentes estilos y términos
  • Botones cambian de lugar
  • Colores diferentes sin razón
  • Términos confusos

2. Feedback Informativo

Retroalimentación Continua

El sistema debe proporcionar feedback para cada acción del usuario

Buen Feedback
Archivo guardado exitosamente
Progreso: 75%
  • Confirmación de acciones
  • Indicadores de progreso
  • Mensajes de error claros
Sin Feedback
(Sin respuesta visual...)
¿Qué está pasando?
  • Botones sin respuesta
  • Procesos silenciosos
  • Errores sin explicación

3. Prevención de Errores

Diseño Preventivo

El sistema debe diseñarse para que los usuarios no puedan cometer errores graves

Prevención Efectiva
Email válido
¿Confirmar?
Eliminado de forma segura
  • Validación en tiempo real
  • Confirmaciones para acciones críticas
  • Campos con formato automático
Sin Prevención
(Sin validación)
¡Eliminado sin confirmación!
¿Era importante?
Sin confirmación ni validación - ¡Peligroso!
  • Eliminación sin confirmación
  • Campos sin validación
  • Acciones irreversibles fáciles

4. Usabilidad Universal

Reconocer la Diversidad

Reconocer las necesidades de usuarios diversos (novatos/expertos, edades, discapacidades)

Diseño Inclusivo
Vista:
Nuevo Documento
Adaptable a diferentes usuarios
  • Múltiples niveles de detalle
  • Opciones de accesibilidad
  • Interfaces adaptables
  • Soporte para diferentes dispositivos
Diseño Excluyente
Configuración Avanzada de Documento
Solo para usuarios expertos
  • Solo para usuarios expertos
  • Sin opciones de accesibilidad
  • Interfaz rígida
  • Asume conocimiento previo

5. Diálogos con Cierre

Secuencias Completas

Las secuencias deben tener principio, desarrollo y final claros

Cierre Claro

3. ¡Completado!

Secuencia clara: Inicio → Progreso → Final
  • Confirmación de finalización
  • Resumen de acciones realizadas
  • Indicadores de progreso
  • Estado final evidente
Sin Cierre
Procesando...
Trabajando...
Estado desconocido...
Proceso terminado
¿Se completó correctamente?
¿Dónde está el resultado?
¿Puedo continuar?
Usuario confundido sobre el estado
  • Procesos que "se cuelgan"
  • Sin confirmación final
  • Usuario no sabe si terminó
  • Ambigüedad en el estado

Resumen: Las 8 Reglas de Oro

Reglas 1-4

  1. Consistencia - Uniformidad visual y funcional
  2. Usabilidad Universal - Diseño inclusivo
  3. Feedback Informativo - Retroalimentación continua
  4. Diálogos con Cierre - Secuencias completas

Reglas 5-8

  1. Prevención de Errores - Diseño preventivo
  2. Reversibilidad - Fácil deshacer
  3. Locus de Control Interno - Usuario en control
  4. Reducción de Carga de Memoria - Pantallas simples

Para PE-1.1: Enfócate en las primeras 3 reglas (Consistencia, Feedback, Prevención)

Para el examen: Debes conocer las 8 reglas completas

Metodología de Trabajo

Proceso de Análisis

  1. Selección: App móvil popular
  2. Análisis: Consistencia visual
  3. Evaluación: Feedback del sistema
  4. Identificación: Prevención de errores
  5. Documentación: Capturas y ejemplos

Criterios de Evaluación

  • 40%: Aplicación correcta de principios
  • 30%: Identificación de problemas
  • 20%: Calidad de capturas
  • 10%: Claridad en presentación

Tips para el Análisis

Consistencia

  • Compara pantallas similares
  • Revisa colores y tipografías
  • Verifica posición de elementos

Feedback

  • Prueba botones y enlaces
  • Observa indicadores de carga
  • Revisa mensajes de error

Prevención

  • Busca validaciones de formularios
  • Identifica confirmaciones
  • Prueba acciones críticas
